# 통합 DB 모니터링 시스템 구축 제안서 목차

## 1. 제안 개요

### 1.1 제안 배경

사내 주요 업무 시스템은 MSSQL, Oracle, Azure SQL Database 등 다양한 데이터베이스 환경 위에서 운영되고 있다. 데이터베이스 성능 저하는 애플리케이션 응답 지연, 배치 처리 지연, 장애, 서비스 중단으로 이어질 수 있으며, 최종 사용자 경험과 업무 연속성에 직접적인 영향을 준다.

현재 DBMS별 관리 도구와 운영 방식이 분산되어 있을 경우 통합적인 장애 징후 탐지, 성능 이력 분석, 담당자 알림, 보안 점검이 어렵다. 따라서 이기종 데이터베이스를 하나의 체계로 모니터링하고, 실시간 장애 예방과 성능 개선을 지원하는 통합 DB 모니터링 시스템 구축이 필요하다.

### 1.2 제안 목적

- 사내 주요 데이터베이스의 실시간 상태와 성능 이력을 통합 관측한다.
- 장애 징후, 병목 쿼리, 락, 교착상태, 리소스 과다 사용을 조기에 탐지한다.
- MSSQL Query Store와 유사한 SQL 성능 이력 분석 체계를 이기종 DB 환경으로 확장한다.
- 성능 저하 원인과 개선 방향을 자동으로 제시하여 DBA와 운영자의 분석 시간을 단축한다.
- 계정, 권한, 보안 설정, 접근 이력을 상시 점검하여 운영 리스크를 낮춘다.
- 실시간 알림과 이슈 관리를 통해 장애 대응 절차를 표준화한다.

### 1.3 제안 범위

- MSSQL, Oracle, Azure SQL Database 대상 통합 모니터링
- 실시간 성능 지표, 세션, 락, 교착상태, Wait, SQL 실행 현황 수집
- 성능 이력, Top SQL, 실행 계획 변경, 성능 회귀 분석
- 계정, 권한, 보안 설정, 접속 이력 모니터링
- 임계치 및 이상 탐지 기반 실시간 알림
- 이슈 관리, 조치 이력, 정기 리포트
- 웹 기반 대시보드, 메뉴 관리, 권한 관리, 시스템 관리

## 2. 현황 및 문제점

### 2.1 운영 환경 현황

- On-premise MSSQL 운영
- On-premise Oracle 운영
- Azure PaaS형 MSSQL 운영
- 업무 시스템별 DB 중요도와 담당자 체계 상이
- DBMS별 성능 분석 도구와 운영 절차 분산

### 2.2 주요 문제점

- DB 장애 징후를 사전에 통합적으로 확인하기 어렵다.
- 세션, 락, 교착상태, 장시간 실행 쿼리 발생 시 즉각적인 원인 파악이 어렵다.
- 과거 성능 이력과 현재 상태를 연결한 분석 체계가 부족하다.
- 실행 계획 변경, 통계 변경, 인덱스 영향으로 인한 성능 저하를 체계적으로 추적하기 어렵다.
- 계정과 권한 변경 이력이 DB별로 분산되어 보안 감사가 어렵다.
- 장애 발생 후 담당자 전파와 조치 이력 관리가 표준화되어 있지 않다.

## 3. 구축 방향

### 3.1 통합 관측성 확보

이기종 DBMS의 성능 지표를 공통 모델로 정규화하여 업무 시스템, DBMS, 인스턴스, 세션, SQL 단위로 추적한다.

### 3.2 실시간 장애 예방

CPU, 메모리, I/O, Wait, Blocking, Deadlock, Long Query, Connection, 로그 사용률 등을 실시간으로 수집하고 임계치와 패턴 기반으로 이상 징후를 탐지한다.

### 3.3 SQL 중심 성능 분석

Top SQL, 실행 계획, 리소스 사용량, 수행 시간 변화, 실행 빈도 변화를 장기 보관하여 성능 저하 원인을 SQL 단위로 분석한다.

### 3.4 운영 및 보안 관리 통합

계정, 권한, 접속 실패, 비정상 접속, 감사 설정 등 보안 관리 항목을 함께 수집하여 성능 관리와 운영 통제를 하나의 시스템에서 수행한다.

### 3.5 자동화와 표준화

알림, 이슈 등록, 담당자 배정, 조치 이력, 리포트 발송을 자동화하여 장애 대응과 운영 보고를 표준화한다.

## 4. 목표 시스템 개요

### 4.1 시스템명

통합 DB 모니터링 및 성능 분석 시스템

### 4.2 시스템 구성

- 데이터 수집 모듈: DB별 성능, 세션, SQL, 보안 데이터 수집
- 데이터 처리 모듈: 지표 정규화, 이상 탐지, 이벤트 생성
- 데이터 저장소: 시계열 성능 데이터, SQL 이력, 알림 이력, 보안 이력 저장
- 분석 엔진: 병목 분석, 성능 회귀 탐지, 실행 계획 변경 분석, 개선 권고
- 웹 포털: 대시보드, 실시간 모니터링, 분석, 리포트, 관리 기능 제공
- 알림 연동: Email, SMS, Teams, Slack, 사내 메신저 연동

### 4.3 사용자 유형

- DBA: 성능 분석, SQL 튜닝, 장애 대응
- 시스템 운영자: 업무 시스템별 DB 상태 확인, 알림 대응
- 보안 담당자: 계정, 권한, 접속 이력, 감사 항목 점검
- 관리자: 시스템 설정, 사용자 및 권한 관리, 운영 리포트 확인
- 개발자: 담당 시스템의 문제 SQL과 성능 저하 원인 확인

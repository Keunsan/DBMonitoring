## 5. 수집 범위 (수집 항목 §11)

### 5.1 1차 수집 DBMS

| DBMS | 1차 구현 | 1차 범위 |
|------|----------|----------|
| **MSSQL** | **완전 구현** | Agentless, §11.1 1차 우선 항목 전부 |
| Oracle | 인터페이스·스텁만 | V$SESSION/V$SQL 등은 T-017, 라이선스 검토 후 |
| Azure SQL Database | 인터페이스·스텁만 | Azure Monitor + DMV는 T-018 |

### 5.2 MSSQL 1차 우선 수집 항목 (§11.1)

| 카테고리 | 포함 | 수집 주기(권장) |
|----------|------|-----------------|
| 연결 상태·가용성 | ○ | 30초~1분 |
| CPU, Memory, I/O, Session | ○ | 10초~1분 |
| Wait | ○ | 10초~1분 |
| Blocking | ○ | 5~30초 |
| Deadlock | ○ | 이벤트 기반 |
| Long Running Query | ○ | 30초 |
| Top SQL (Query Store 또는 DMV) | ○ | 1~5분 |
| 알림 이벤트 원천 지표 | ○ | 정책 연동 |

**주요 DMV/원천:** `sys.dm_exec_requests`, `sys.dm_exec_sessions`, `sys.dm_os_waiting_tasks`, `sys.dm_tran_locks`, Query Store, Deadlock XML.

**비고:** Query Store 비활성 DB는 DMV 기반으로 대체 수집.

### 5.3 1차 제외 수집

- Oracle ASH/AWR (라이선스 검토 전 보류)
- Azure Resource Graph 전체, 방화벽 규칙 변경 감시 (3차 보안)
- 실행 계획 상세 Operation, 인덱스/통계 변경 이력 (2차)
- 계정·권한 전수 (1일 1회 → 3차)

# 통합 DB 모니터링 시스템 화면 구성안

## 1. 화면 구성 원칙

### 1.1 기본 방향

- 전체 DB 상태를 먼저 보여주고, 문제 발생 시 DB, 세션, SQL, 실행 계획, 이슈 순서로 상세 분석할 수 있도록 구성한다.
- 실시간 모니터링 화면과 과거 분석 화면을 분리하되, 동일한 이벤트에서 상호 이동할 수 있게 한다.
- DBA, 운영자, 보안 담당자, 관리자 등 사용자 역할에 따라 필요한 메뉴와 데이터만 노출한다.
- 장애 상황에서는 원인 후보, 영향 범위, 담당자, 조치 상태를 빠르게 파악할 수 있어야 한다.

### 1.2 화면 흐름

1. 통합 대시보드에서 전체 위험 상황 확인
2. DB 상세 화면에서 인스턴스 상태 확인
3. 세션 또는 Wait 화면에서 병목 구간 확인
4. SQL 분석 화면에서 문제 SQL과 실행 계획 확인
5. 이슈 화면에서 담당자 지정 및 조치 이력 관리
6. 리포트 화면에서 기간별 현황 공유

## 3. 주요 화면 상세

## 3.1 통합 현황 대시보드

### 목적

전체 DB의 현재 상태와 위험 요소를 한 화면에서 확인한다.

### 주요 구성

- 전체 DB 수
- 정상, 주의, 경고, 장애 DB 수
- DBMS별 상태 분포
- 업무 시스템별 상태
- 최근 24시간 알림 현황
- 미처리 이슈 현황
- 리소스 사용률 상위 DB
- Long Running Query 상위 DB
- Blocking 발생 DB
- Deadlock 발생 DB

### 주요 필터

- 업무 시스템
- DBMS
- 운영 구분
- 중요도
- 담당자
- 상태

### 주요 동작

- DB 상태 카드 클릭 시 DB 상세 화면 이동
- 알림 클릭 시 알림 상세 화면 이동
- 이슈 클릭 시 이슈 상세 화면 이동

## 3.2 업무 시스템별 현황 화면

### 목적

업무 서비스 관점에서 DB 상태와 영향 범위를 확인한다.

### 주요 구성

- 업무 시스템 목록
- 업무 중요도
- 연결 DB 목록
- 업무별 장애 위험도
- 업무별 최근 알림
- 업무별 미처리 이슈
- 담당자 정보

### 주요 동작

- 업무 시스템 선택 시 연결 DB 목록 표시
- DB 선택 시 DB 실시간 현황으로 이동

## 3.3 DB 실시간 현황 화면

### 목적

특정 DB 인스턴스의 현재 성능 상태를 실시간으로 확인한다.

### 주요 구성

- 수집 상태
- CPU 사용률
- 메모리 사용률
- 디스크 I/O
- 네트워크 I/O
- 세션 수
- Active Session 수
- TPS 또는 Batch Requests
- 로그 사용률
- Temp 영역 사용률
- DB별 주요 Wait
- 최근 알림

### 주요 그래프

- CPU/Memory 실시간 추이
- I/O 실시간 추이
- Active Session 추이
- Wait 추이
- 로그 사용률 추이

### 주요 동작

- 세션 수 클릭 시 실시간 세션 화면 이동
- Wait 클릭 시 Wait 상세 화면 이동
- 알림 클릭 시 알림 상세 화면 이동

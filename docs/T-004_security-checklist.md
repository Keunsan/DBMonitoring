## 2. 보안 기본 원칙

| 원칙 | 적용 기준 |
|------|-----------|
| 서버 측 검증 | 모든 입력, 필터, DB 연결 설정, 관리 액션은 API/서버에서 검증 |

## 6. 금지 사항

- DB 접속 문자열, 비밀번호, API Key, SSO token을 코드·문서·로그에 남기지 않습니다.
- SQL 문자열을 사용자 입력과 직접 결합하지 않습니다.
- 권한 없는 데이터를 API에서 내려준 뒤 클라이언트에서 숨기는 방식은 금지합니다.
- 운영 응답에 stack trace, 내부 경로, 원본 쿼리, 접속 정보를 노출하지 않습니다.
- 세션 종료, 권한 변경, DB 등록/삭제 등 고위험 작업을 감사 로그 없이 구현하지 않습니다.

## 7. 1차 MVP 보안 완료 기준

아래 항목이 충족되어야 1차 MVP 완료 기준에 포함할 수 있습니다.

- [ ] SSO 또는 개발용 mock 인증 플로우가 있고 보호 라우트/API가 동작합니다.
- [ ] 역할별 메뉴/기능/API 권한 검증이 서버에서 수행됩니다.
- [ ] DB 접속 정보가 평문으로 저장되지 않습니다.
- [ ] SQL Text는 기본 마스킹 또는 normalized_sql로 저장됩니다.
- [ ] 주요 관리자 액션이 `AUDIT_LOG`에 기록됩니다.
- [ ] `npm run lint`가 통과합니다.
- [ ] 주요 화면에 로딩/빈/오류/권한 없음 UI가 있습니다.
